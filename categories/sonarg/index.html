Subquery in a different database

August 29, 2020
SonarW, SonarG
By default subqueries are executed in the same database as the original query. This behaviour can be changed by appending database name in front of collection name: { "$ns" : "databaseName.collectionName", "$q" : {}, "$p" : "" } This is particularly useful when group_members collection needs to be referenced when working outside of sonargd database: { "$ns" : "sonargd.group_members", "$q" : { "Group Type" : "<Enter Group Type>", "Group Description" : "<Enter Group Description>" }, "$p" : "Group Member" }
